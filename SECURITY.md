# Security Policy

## Supported Versions

Security fixes are applied on:

| Version | Supported |
| --- | --- |
| `master` | Yes |
| Latest tagged release | Yes |
| Older releases | No |

If a fix is required, reproduce it against the current `master` branch whenever possible.

## Reporting a Vulnerability

Do not open a public GitHub issue for a security vulnerability.

Use GitHub private vulnerability reporting for this repository when available. If private reporting is not available in your context, contact the repository maintainers directly through the organization owners and include:

- affected version, branch, or commit
- vulnerability type and impact
- reproduction steps or proof of concept
- any proposed mitigation or fix

Please avoid publishing exploit details until a fix is available.

## What to Expect

- We will triage the report and assess impact.
- We may ask for clarification or reproduction details.
- Confirmed issues will be fixed on supported versions only.
- A coordinated disclosure timeline will be agreed when needed.
